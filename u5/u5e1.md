# T1. Uso avanzado de funciones

## Unidad 5. Closures

## Ejercicio 1.5.1 (Castellano)

Uno de los usos más habituales históricamente en JavaScript para sacar
partido del concepto de `closure` es el de crear expresiones de funciones ejecutadas de forma inmediata (o funciones auto ejecutables), que encapsulan toda su lógica de manera que evitan cualquier conflicto con el contexto exterior en el que se ejecutan.

Esto era especialmente importante cuando no disponíamos de `const` o
`let` para declarar variables a nivel de bloque, por lo que los conflictos en variables declaradas a través de `var` eran mucho más habituales.

Estas funciones se conocen como [funciones IIFE](https://developer.mozilla.org/es/docs/Glossary/IIFE)
 (Immediately invoked function expression) y puedes encontrar más
 información sobre ellas aquí:

https://developer.mozilla.org/es/docs/Glossary/IIFE

En este ejercicio vamos a practicar el uso de este tipo de funciones, que es muy útil para encapsular nuestro código.

### A. Función `generate`

Crea una función denominada `generate` que recibe como único parámetro
una cadena de texto `text` y que:

- Devuelve `false` si no se pasa una cadena de texto como parámetro.

- Si recibe una cadena de texto como parámetro la pasa a minúsculas
y devuelve la cadena resultante de convertir las vocales por su
equivalente en número siguiendo la siguiente tabla:

| vocal | dígito |
| :-: | :-: |
| a, á | 1 |
| e, é | 2 |
| i, í | 3 |
| o, ó | 4 |
| u, ú, ü  | 5 |


### B. Convierte dicha función en IIFE

Envuelve esa función `generate` en una IIFE de manera que esa IIFE retorne la referencia a la función `generate`, y almacene esa referencia en una variable `converter`. Es decir:

```js
const converter = // IIFE que devuelve referencia a generate
```

De manera que posteriormente seamos capaces de invocar este código:

```js
let result = converter('Hola!');
// Expected result: h4l1!
```

# T1. Ús avançat de funcions

## Unitat 5. Closures

## Exercici 1.5.1 (Català)

Un dels usos més habituals històricament a JavaScript per treure
partit del concepte de `closure` és el de crear expressions de funcions
executades de forma immediata (o funcions autoexecutables), que
encapsulen tota la seva lògica de manera que eviten qualsevol conflicte amb el context exterior en què s'executen.

Això era especialment important quan no disposàvem de `const` o
`let` per declarar variables a nivell de bloc, de manera que els conflictes en variables declarades a través de `var` eren molt més habituals.

Aquestes funcions es coneixen com a [funcions IIFE](https://developer.mozilla.org/es/docs/Glossary/IIFE)
  (Immediately invoked function expression) i en pots trobar més
  informació sobre elles aquí:

https://developer.mozilla.org/es/docs/Glossary/IIFE

En aquest exercici practicarem l'ús d'aquest tipus de funcions, que
és molt útil per encapsular el nostre codi.

### A. Funció `generate`

Crea una funció anomenada `generate` que rep com a únic paràmetre
una cadena de text `text` i que:

- Retorna `false` si no es passa una cadena de text com a paràmetre.

i retorna la cadena resultant de convertir les vocals pel seu
equivalent en nombre seguint la taula següent:

| vocal | dígit |

```js
const converter = // IIFE que torna referència a generat
```

De manera que posteriorment siguem capaços d'invocar aquest codi:

```js
let result = converter('Hola!');
// Expected result: h4l1!
```
